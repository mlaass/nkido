## akkado/ — 4 commits, ~17 files touched

- **Add beat(n) helper to stdlib** (`55cc4dae`)  
  Convenience wrapper around `trigger(1/n)` for beat-rate phasors; adds `stdlib.hpp`.

- **Accept sample-name string in sample()/sample_loop() third arg** (`10c880f5`)  
  The third arg was previously a numeric sample-bank ID (effectively unusable). Now `sample(trig, pitch, "bd")` registers the name in the required-samples ledger; emits a `PUSH_CONST` placeholder patched by the host at load time. Supports `"bd:3"` and `"Bank/name:variant"` forms; numeric IDs remain valid for back-compat. Updates `builtins.hpp`, `codegen.cpp/hpp`, `required_sample.hpp`, `akkado.cpp`, and adds 84-line test coverage in `test_codegen.cpp`.

- **Make Pattern values own their sample requirements** (`752ece6b`)  
  `PatternPayload` now carries a value-type `sample_refs` vector. Every Pattern producer propagates it via a new `publish_sample_refs` helper, replacing the previous inline-loop approach in `handle_bank_call`/`handle_variant_call` that silently dropped registrations for wrapped transforms (`.fast`, `.slow`, etc.). The web loader's legacy fallback path is removed; all samples use the bank-aware path. Adds invariant test asserting metadata survives arbitrary transform stacks and never leaks between independent chains.

- **Wire SAMPLE_PLAY in pattern-transform handlers for sample patterns** (`1e45d3f4`)  
  Five of six codegen sites that re-emit the `SEQPAT` pipeline dropped the `PUSH_CONST(1.0) + SAMPLE_PLAY` pair for sample patterns, returning raw sample-IDs (DC) instead of audio. Extracts `emit_sampler_wrapper` and calls it from all six sites: `handle_mini_literal`, `handle_pattern_reference`, `handle_velocity_call`, `handle_bank_call`, `handle_variant_call`, `handle_transport_call`. Strengthens `test_codegen.cpp` to assert `SAMPLE_PLAY` is present in bytecode.

## cedar/ — 1 commit, 1 file touched

- **Fix serve audio dropout after stop; redirect VM stdout to stderr** (`ef96e12a`)

## tools/ — 4 commits, ~11 files touched

- **Add nkido-cli serve mode and .akk extension recognition** (`91ecceb3`)  
  New headless persistent subcommand `nkido-cli serve` reads newline-delimited JSON commands from stdin and emits events to stdout (`load`/`stop`/`set_param`/`quit` → `ready`/`compiled`/`diagnostic`/`stopped`/`param_changed`/`error`). Reuses `AudioEngine` and Cedar's hot-swap path for glitch-free live coding without respawning the CLI. Also makes `detect_input_type` recognize `.akk` files explicitly. Adds `serve_mode.cpp/hpp` (~442 lines).

- **Add SDL window with live waveform and level meter to serve mode** (`5f0653cc`)  
  Opens a 480×200 graphical window when `serve` starts for visual confirmation that audio is being produced. Main thread runs the SDL event loop; a worker thread owns `std::cin` and pushes `SDL_USEREVENT`s (thread-safe). Factors command dispatch into `handle_command_line()`. Window failure is non-fatal — falls back to headless mode.

- **Fix nkido-cli serve silence and misleading sin(440) help example** (`09e08f49`)  
  Defers `engine->init()` until the first `load` command arrives, mirroring `play` mode's open/load/unpause sequence and avoiding PulseAudio/PipeWire stream corking. Fixes `--help` example from `sin(440)` (math sine of radians, sounds like DC) to `sine(440)`.

- **Log SDL audio driver/format and callback first-fire** (`25d4f6d7`)  
  Adds two stderr diagnostics to `audio_engine.cpp`: the negotiated SDL device info after `SDL_OpenAudioDevice`, and a one-shot log on the first `audio_callback` invocation. Enables pinpointing silence to device-open failure, callback not firing, or VM producing zeros.
