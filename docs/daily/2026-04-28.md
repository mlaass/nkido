All five phases (A‚ÄìE) of the pattern-array-note-extensions Phase 2 PRD were

## Changes by area

### akkado/ ‚Äî 15 files

- **Phase A1 ‚Äì `early` / `late` / `palindrome` / `compress`** (`0c5bcdd`)
  Four compile-time event-list rewrites; drops the `compress`‚Üí`comp`
  audio-compressor alias to free the canonical pattern-transform name.

- **Phase A2 ‚Äì `ply` / `linger` / `zoom` / `segment`** (`d0fe5cb`)
  `zoom` clips events to a `[s,e)` window and stretches to fill the cycle;
  `segment` resamples at n evenly-spaced points; `linger` keeps the first
  fraction and loops; `ply` repeats each event n times within its slot.

- **Phase A3 ‚Äì `swing` / `swingBy`** (`520a058`)
  Slice-based groove transforms; `swingBy` accepts a custom amount and grid
  size, `swing` defaults to 1/3 swing on a 4-slice grid.

- **Phase A4 ‚Äì `iter` / `iterBack`** (`6e2fd06`)
  Cycle-index-driven rotation compiled into Cedar's `SequenceState`; the
  rotation is applied at runtime so patterns shift forward or backward by one
  step per cycle. Cedar changes listed separately below.

- **Phase B ‚Äì `run` / `binary` / `binaryN`** (`fa8d80b`)
  Algorithmic constructors that emit event streams from numeric arguments:
  `run(n)` produces 0..n‚àí1; `binary(n)` / `binaryN(n,b)` derive trigger
  patterns from the binary representation of n. Composable via dot-call.

- **Phase C ‚Äì voicing system** (`e398fadc`)
  New `akkado/voicing.{hpp,cpp}` module with greedy voice-leading algorithm.
  New built-ins: `anchor`, `mode` (Below/Above/Duck/Root), `voicing`,
  `addVoicings`. Built-in voice dictionaries: `close`, `open`, `drop2`,
  `drop3`. Chord progressions can now route through smooth voice-leading
  before emission as multi-value pattern events.

- **Phase D0 ‚Äì velocity-shorthand fix** (`9a38e30`)
  `pattern_eval.cpp` now correctly copies per-atom velocity from
  `MiniAtomData` into `PatternEvent.velocity`; previously the `:0.x` suffix
  was discarded and events always had velocity 1.0.

- **Phase D1+D2 ‚Äì record-suffix mini-notation** (`1a14690b`)
  `c4{vel:0.8, dur:0.5}` syntax wired end-to-end: lexer `try_lex_record_suffix()`,
  AST `properties` vectors on pitch/sample/chord tokens, codegen applying `vel`
  and `dur` to event fields. Unrecognised keys (`bend`, `cutoff`, etc.) are
  preserved on `atom_data.properties` for a future cedar runtime extension.
  Polymeter `{a b}%n` and whitespace-separated `c4 {a b}%3` are
  unambiguously distinguished.
