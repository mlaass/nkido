## cedar/ — 17 files touched

**Smooch: implement OSC_WAVETABLE wavetable oscillator (multi-bank)**
(`efe84eb`)

A Serum/Vital-class wavetable oscillator added end-to-end. Offline
preprocessor (`cedar/src/wavetable/preprocessor.cpp`) applies DC removal,
phase alignment, raised-cosine spectral taper, and IFFT/RMS normalization to
build an 11-level mip pyramid. Audio-time opcode `op_osc_wavetable` uses
4-point Niemitalo-optimal interpolation with equal-power crossfade across mip
octave boundaries. Multi-bank registry (up to 64 banks) keyed by name → ID;
bank ID packed in `inst.rate`. 9 new unit tests in `cedar/tests/test_wavetable.cpp`.
Two PRD bugs corrected: `bin >= cutoffBin` → `bin > cutoffBin` (would zero
the fundamental); mip selection denominator 1024 → 2048 (prevents aliasing at
10 kHz on sawtooth).

## akkado/ — ~25 files touched

**Smooch: implement OSC_WAVETABLE wavetable oscillator (multi-bank)**
(`efe84eb`)

New `wt_load` compile-time directive records `RequiredWavetable{name, path, id}`
in the compile result without emitting an instruction. `smooch` builtin
special-handled in codegen (mirrors soundfont): string-literal first arg
resolved to bank ID at compile time, E199 on missing `wt_load`. Bank IDs
assigned in source order so the runtime registry must be cleared before
loading. Adds `codegen_patterns.cpp` for pattern codegen used by this opcode.

**Patterns as scalar values: typed prefixes + auto-coerce** (`e28cb0f`)

Patterns are now first-class values usable anywhere a scalar Signal is
accepted. Four typed mini-notation string prefixes added: `v"…"` (value),
`n"…"` (note), `s"…"` (sample), `c"…"` (curve), joining the existing
`p"…"` / `t"…"` forms. `MiniParseMode` enum replaces the old
`sample_only`/`curve_mode` booleans throughout the lex/parse pipeline. Value
mode adds a numeric-atom path (signed decimals, scientific notation) and
rejects letter-leading atoms with E163. `PatternPayload` gains
`is_sample_pattern` + `max_voices`; `args_are_signal` flag on `BuiltinInfo`
gates Pattern→Signal coercion; polyphonic non-sample patterns in scalar slots
reject E160. New builtins: `value()` / `note()` / `scalar()` (idempotent on
Signal, errors E161 on sample/poly). Phase 2.1 §11.2 bend/aftertouch/dur
transforms accept Pattern as second argument via SEQPAT_PROP. 539/539 akkado
tests pass; 6 new cases in `test_pattern_scalar.cpp`.
