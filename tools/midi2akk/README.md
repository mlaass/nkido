# midi2akk

Convert MIDI files to Akkado programs. Every track is rendered as a
`pat(...) |> soundfont(%, "GM.sf3", <program>)` line, and all tracks are
summed and scaled by `1/N` into the stereo output.

## Usage

```bash
cd tools/midi2akk
uv sync

# Write tune.akk next to the input
uv run python -m midi2akk tune.mid

# Pipe to stdout
uv run python -m midi2akk tune.mid -o -

# Tight 32nd-note grid, allow up to 6 voices per polyphonic track
uv run python -m midi2akk tune.mid --subdivision 32 --max-voices 6

# Use the saw+ADSR fallback instead of the soundfont path
uv run python -m midi2akk tune.mid --instrument osc

# Point at a specific soundfont filename the host has loaded
uv run python -m midi2akk tune.mid --soundfont FluidR3Mono_GM.sf3
```

The emitted source references a soundfont by name only. The Akkado host is
responsible for loading it — the web app already ships
`web/static/soundfonts/TimGM6mb.sf3` and `FluidR3Mono_GM.sf3`, so pick a
filename from there (or pass `--soundfont`) and make sure the same file is
registered at runtime.

## Output shape

```akkado
// Generated by midi2akk from band.mid
// Tempo: 100.00 bpm, time signature: 4/4, bars: 1, subdivision: 1/16
// Backend: soundfont (GM.sf3)

bpm = 100

// Piano Lead — program 0: Acoustic Grand Piano
piano_lead = pat("[c4 _ _ _ e4 _ _ _ g4 _ _ _ c5 _ _ _]")
  |> soundfont(%, "GM.sf3", 0)

// Bass — program 33: Electric Bass (finger)
bass = pat("[c2 _ _ _ c2 _ _ _ e2 _ _ _ c2 _ _ _]")
  |> soundfont(%, "GM.sf3", 33)

// Drums — GM drum kit
drums = pat("[c2 _ _ _ d2 _ _ _ c2 _ _ _ d2 _ _ _]")
  |> soundfont(%, "GM.sf3", 128)

// Mix down to stereo with even gain per track
(piano_lead + bass + drums) * (1.0 / 3) |> out(%, %)
```

Polyphonic tracks are split into monophonic voices (`<name>_v1`, `<name>_v2`,
...) up to `--max-voices`. Drum tracks (MIDI channel 10) use preset 128 —
the SF2/SF3 percussion bank — and keep their original note numbers so the
soundfont maps them to drum sounds internally.

## Limitations

These come from the mini-notation grid and Akkado's current pattern model.
Most are deferrable — flagged so no one is surprised by lossy round-trips.

- **Velocity is dropped.** Mini-notation has no velocity slot; every note
  plays at the pattern's default velocity.
- **Tempo and time signature are frozen** at their first values. Mid-song
  tempo maps, meter changes, and ritardandi are ignored.
- **No pitch bend, aftertouch, or CC automation.**
- **Quantize-up on short notes.** Anything shorter than one slot is bumped
  to one slot; two onsets that snap to the same slot drop the second (with
  a comment in the output noting how many notes were lost).
- **Voice cap.** Dense polyphony beyond `--max-voices` drops the overflow
  notes (also reported in the output comments).

```bash
uv sync --group dev
uv run pytest
```

Tests live in `tests/` and build their fixtures in-memory with `mido` — no
